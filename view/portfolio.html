Последняя работа

Проект Ipricer.ru - Система консолидированного управления прайс-листами, предназначена для сборки, аналитики прайс-листов автозапчастей. Распространяется свободна и может быть интегрирована по средствам Web Api в любую другую систему например магазин автозапчастей.

При написании системы использовались следующие технологии со стороны фронтенда: Героический фреймворк ангуляр 2,0 -RC-6, за время проекта мигирировали до 2.4.1, система полностью готова к миграции на ангуляр 4,0. Сборка основана на system.js, npm + gulp. Использованы библиотеки: d3 ( графики и чарты), lodash ( без него никуда )), ng2-dnd (драгать и дропать во все стороны), primeng ( библиотека компонентов на angular 2+ gulp-jsdoc3 ( построена документация проекта), gulp-rigger ( для конкотинации скриптов), jasmine и karma ( для тестирования).

Верстка на основе bootstrap v4.0.0-alpha.5, использованы material-design-icon ("mdi": "1.7.22"), bower, compass ( фреймворк для sass), шрифты семейства openSans

Так как на момент запуска angular-cli был совсем сырой и официальная документация рекомендоваоа system.js на нем был запущен проект. Но мне не хватало консольных возможностей angular-cli, в результате были написаны bush скрипты, заменяюшие cli ( они могут генерировать компоненты в нужном месте соблюдая иерархию) а также могут генерировать сразу бандлы ( состоящие из комопнентов создания, изменения и отображения списка, сервиса, конфигурационного файла, и файла документации)

Система реализована как REST, использованы патерны проектирования одиночка, фабричный метод, билдер. Компоненты, зависимы между собой через dependensy injection в конструторе, свойства и по средствам методов. Архитектура прямо скажем удалась не очень, нужно делать хороший рефакторинг но обраружить проблему это на 50% решить. Используется оргомное количество ивентов. Возможности ангуляр 2,4 использованы на 90% ( 10% это валидация форм средствами ангуляра - не использовалось из за большого количества вариантов и дополнительных функций). Тайп-скрипт стал роднее и ближе чем js, вообще сомневаюсь что можно писать по другому. Ангуляр дает очень приятные инструменты для разделения кода, изоляция компонентов в плане css дает возможность многократноиспользовать код, реализовать все прелести ООП и реализовать полноценный MVC функционал. Все что написано со стороны фронтенда, написано мной. (43 бандла это примерно 1300 файлов)

Ereport

В марте 2016 года, мне посчастливилось разработать простую систему для сдачи отчетов в электронном виде, клиентами компании "Интеко Системс" В данной работе были интересны следующие моменты: Верстка системы осуществлялась на основе фреймвока bootstrap4 (ожидает релиза в ближайшем будующем но уже доступном на официальном сайте), помимо новейшего инструмента стартовая страница имеет ряд особенностей, по первых тут используется горизонтальный скролинг, который накладывает свои ньюансы на адаптивность, несмотря на это система поддерживатеся всеми современными браузерами в том числе IE (IE10+).

Система имеет 3 вида навигации, использует интересную svg анимацию на библиотеке lazilinepainter для отображения этапов работы и логотипа. При помощи уже привычного jquery осуществляется анимация необходимых элементов и ротация строниц. Внутренняя архитектура построена на основании фреймвока symfony 2.7 (doctrina, twig, yml, symfony) Сборка бекенда и зависимости при помощи composer. Использованы бандлы "fosuserbandle". Система позволяет скачивать заготовки, загружать отчеты. ведет разграничение пользователей и их прав. Имеет систему публикации внутренних новостей. Для работы системы используются javascript editor ckEditor. Сборка фронтенда при помощи gulp, пакетные менеджеры npm, bower. Иконки fontawesome. система контроля версий git. (проект полностью выполнен мной)
